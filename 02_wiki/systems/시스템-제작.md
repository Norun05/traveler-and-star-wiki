# 시스템 - 제작

## 0) 목적

제작 시스템은 재료를 모아 도구, 가구, 소모품 등을 만드는 시스템이다.

요리 시스템과 달리 정해진 레시피 기반으로 작동한다. 창의적인 조합이 아니라 정확한 재료와 절차가 필요하다.

---

## 1) 두 가지 모드

### 1.1 실험 모드

레시피를 모르는 상태에서 재료를 조합해 시도하는 모드다.

- 재료 슬롯에 재료를 넣고 실험 실행
- 시간 소모 있음 (현재 코드 기준 10분, 추후 조정 가능)
- 성공 시: 레시피 발견 + 결과물 획득
- 실패 시: 재료 전량 환급, 시간만 소모

실험 모드는 레시피 탐색 수단이다. 성공률 개념이 없고, 재료 조합이 유효한 레시피와 맞는지 여부만 판정한다.

### 1.2 제작 모드

알고 있는 레시피로 제작하는 모드다.

- 레시피 목록에서 선택 후 실행
- 성공률이 적용됨
- 성공 시: 결과물 획득, 필요 재료 소모
- 실패 시: 재료 일부 환급 (전량 소모 아님)

---

## 2) 성공률 구조

제작 성공률은 세 가지 요소의 합산이다.

```
성공률 = 기본 성공률 + 숙련도 레벨 보너스 + 제작 횟수 보너스
```

### 2.1 기본 성공률

제작품 티어에 따라 다르다. 티어가 높을수록 낮다.

- 하위 제작품: 높음
- 중위 제작품: 중간
- 상위 제작품: 낮음

구체적인 수치는 아이템 목록 설계 시 확정.

### 2.2 숙련도 레벨 보너스

제작 숙련도 레벨이 오를수록 보너스가 붙는다. 구체적인 레벨당 수치는 미결정.

### 2.3 제작 횟수 보너스

특정 제작품을 반복 제작할수록 소폭 상승한다. 단, 상한이 있어 무한정 오르지 않는다.

상한을 두는 이유: 무한정 오르면 결국 성공률 100%에 수렴해 긴장감이 사라짐. 숙련자도 상위 제작품에는 일정 리스크가 남는 구조를 유지하기 위함.

구체적인 상한값은 미결정.

---

## 3) 실패 처리

제작 실패 시 재료를 일부 환급한다. 전량 소모는 아니다.

- 전량 소모: 페널티가 너무 가혹해 도전 기피
- 전량 환급: 실패 페널티 없음, 긴장감 없어짐
- 일부 환급: 적당한 중간. 도전은 하되 신중하게.

환급 비율은 미결정.

---

## 4) 제작 결과물 카테고리

레시피는 기획자가 하나씩 직접 설계한다. 요리처럼 로직으로 자동 생성되는 구조가 아니다.

### 1차 가공품

원재료를 가공한 중간 재료다. 다른 레시피의 재료로 쓰인다. 최종 결과물이 아니라 제작 과정의 중간 단계.

예시: 판자, 막대기, 밧줄, 직물 등

### 도구

채집, 조사, 제작 관련 도구. 구식/신식 구분 있음.

구식 도구는 신식 도구의 무조건적 하위호환이 아니다. 도구 효과가 필요 없는 단순 반복 작업에서는 내구도 높은 구식 도구가 실질적으로 더 나은 선택이 된다.

### 가구

하우징용 제작품.

### 소모품

탐색 중 소모하는 것들. 예시: 횃불, 포충제 등.

### 기타

선물용, 판매용 공예품 등.

---

## 5) UI — 중간 재료 퀵크래프트

최종 결과물 레시피를 보는 화면에서, 필요한 1차 가공품을 탭 이동 없이 바로 제작할 수 있는 기능을 넣는 방향을 지향한다.

예시:
- 가구 탭에서 선반 레시피 확인
- 재료 목록에 "판자 x2" 표시
- 판자 옆 퀵크래프트 버튼 클릭 → 판자 제작 화면으로 진입
- 판자 제작 후 자동으로 선반 레시피 화면으로 복귀

1차 가공품 탭으로 수동 이동하는 번거로움을 줄이는 편의 기능이다. Don't Starve Together의 유사 편의 기능을 참고.

---

## 6) [미결정] 슬롯

- 티어 단계 수 및 각 티어별 기본 성공률 수치
- 숙련도 레벨당 보너스 수치
- 제작 횟수 보너스 상한값
- 실패 시 재료 환급 비율
- 실험 모드 시간 소모 (현재 10분 고정, 난이도별 차등 가능성 있음)
- 레시피 발견 후 도감 등록 여부
